## 주요기능
1. 두비티아이 : 사용자의 두더지 유형을 검사합니다.
2. 그룹 매칭 : 사용자 유형과 유사한 유저를 매칭합니다.
3. 미션 : 사용자가 세상 밖에 나올 수 있는 미션을 수행합니다.

<br>

## 컨벤션 규칙과 브랜치 전략

**☑️ 작업 유형**
```
[Feat]     : 새로운 ui 또는 기능 구현
[Chore]    : 타입 및 변수명 변경, 패키지 구조 변경, 버전 코드 수정 등의 작은 작업
[Refactor] : 기능 변경 없는 코드 수정
[Add]      : 부수적인 코드 추가 및 라이브러리 추가, 새로운 파일 생성
[Fix]      : 버그 및 오류 해결
[Hotfix]   : issue나 QA에서 문의된 급한 버그 및 오류 해결
[Init]     : 프로젝트 기초세팅
[Docs]     : 문서 작성 및 수정 (README.md 등)
[Style]    : ui 수정 (폰트 스타일, 컬러 등)
```

<br>

**1️⃣ Branch Convention**

main 브랜치를 default로 합니다

```
작업유형/#이슈번호-작업내용
```

- 예시
    - setting/#2-designsystem-color
    - feat/#30-home-ui
    - refactor/#27-signin-viewmodel

<br>

**2️⃣ Commit Convention**

```
[작업유형/#이슈번호] 내용 작성
```

- 예시
    - [feat/#12] 버튼 컴포넌트 작성

<br>

**3️⃣ Issue Convention**

Issue Title만 작성합니다
    
```
[이슈유형] 이슈 내용
```
- 예시
    - [feat] 바텀 내비게이션 구현

<br>

**4️⃣ PR Convention**

PR Title만 작성합니다
    
```
[PR유형/#이슈번호] PR 내용
```
- 예시
    - [feat/#15] 회원가입 UI 구현

<br>

**5️⃣ Merge Convention**

```
[MERGE] #이슈번호 -> main
```
